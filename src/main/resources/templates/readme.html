系统操作实例（BVT场景用例）

第1步：admin/admin登录系统，新建机构org1, org2, org3；新建人员user1(机构=org1), user2(机构=org2), user3(机构=org3)，user4（机构=org3）；新建项目project1(项目经理=user1, 主办单位=org1, 临床研究单位=org2, org3). 退出。
第2步：user1（项目经理）登录，项目管理-->权限设置，设参试人员=user2, user3,统计员=user4.退出。
第3步：user4(统计员)登录，随机管理-->随机设置，退出。
第4步：user2(参试人员)登录，随机管理-->取号，查看取号结果，退出。
第5步：user4(统计员)登录，随机管理-->揭盲，下载结果到本地，退出。
第6步：admin登录，创建project2，项目经理设置为user3，重置user3的密码，退出。
第7步：user3(参试人员)登录（密码是123456），项目管理-->切换项目，修改密码，修改个人资料，回到首页。

操作细节

1. 1个人员可以在多个项目中有权限，在同一项目中可以有多个普通权限（project manager, doctor, statistician）。
2. 1个项目可以有1个申办单位和多个临床研究单位。
3. 若该项目已经进行了随机设置，不管是否取号，修改项目时临床研究单位列表不可编辑；若未设置过随机，则可以编辑修改。
4. 项目经理的权限是在admin新建、修改项目时指定的，而参试人员、统计员的权限是项目经理在“权限设置”中指定的；新增加的人员激活状态都是“未激活”，不能登录，只有在其被项目经理指定权限后，状态才会变更为“已激活”并可以正常登录。
5. “随机设置”页面，若从未设置过则显示wizard设置项，若已设置显示的是设置的结果。
6. “取号”页面，若随机设置了分层，取号页面会显示分层项，否则不显示。
7. 每个人可以修改个人基本信息或密码，若忘记密码，admin可以重置为密码123456。
8. 最近操作显示的是数据库中最近的20条记录，滚动显示。
